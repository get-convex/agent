Convex Agent core boundary

Agent core owns runs. Apps compose everything around them.

The clean Agent core owns threads, messages, runs, tools, approvals, and Agent event payloads. Stream owns ordered event storage and HTTP serving. Apps own retrieval, auth, rate limits, billing, API keys, files, workflows, and UI/framework protocols.

Rule: Agent should expose durable Agent primitives, not product policy or framework glue.

Before

Agent tries to be the whole application stack.

Coupled
App calls Agent-specific helpers Chat, files, RAG, auth, deltas, and workflows orbit Agent APIs.
|
Agent owns product policy File refcounts, API keys, memories, rate limits, and wrapper protocols leak into core.
|
Agent owns transport machinery WebSocket deltas, streaming messages, and HTTP framing live beside message storage.
|
Hard to compose Apps must accept Agent's storage and protocol choices.

After

Agent is a Convex primitive apps build around.

Composed
App owns policy Auth, API keys, rate limits, billing, RAG, files, and workflows stay in app code.
|
Agent owns durable runs Threads, messages, runs, tools, approvals, and Agent event payloads stay in core.
|
Stream owns event transport Run events use Stream for ordered storage, cursors, idempotent appends, and HTTP.
|
Adapters come later Vercel, TanStack, AI SDK, and React wrappers map onto the run model.

Boundary map

Agent core

Runs and messages

Threads, Agent messages, run lifecycle, tools, approvals, and Agent event payloads.

Moved

Stream transport

Old WebSocket delta tables and SSE loops give way to Stream: events, cursors, appendTail, HTTP.

Moved

Context and RAG

Embedding tables, memories, and hidden retrieval are gone. Apps pass context loaders.

Moved

API keys and auth

Playground API keys and auth policy stay outside Agent. Apps own request authorization.

Moved

Files

Agent no longer owns a files table, refcounts, or file helper APIs. Message parts hold app-owned refs.

Derived

Tool-call cache

Agent derives tool-call state from the run event stream instead of duplicating it on runs.

Composed

Rate limits and billing

Apps enforce quota before start/execute or expose usage through context and run metadata.

Composed

Workflows and UI

WorkflowManager, React, Vercel, TanStack, and AI SDK belong in app code or adapters.

Core API after separation

App systems compose around Agent runs
await rateLimiter.limit(ctx, { key: userId });
await billing.reserve(ctx, { userId });

const run = await supportAgent.runs.start(ctx, {
  threadId,
  userId,
  prompt,
  key: `client-message:${clientMessageId}`,
});

await supportAgent.runs.execute(ctx, {
  runId: run.runId,
  tools,
  context: [loadRagContext, loadFileContext, loadBillingContext],
});

return await supportAgent.http(ctx, request, { runId: run.runId });

Files as one example

Before: Agent-owned files
const blob = new Blob([bytes], { type: mediaType });
const storageId = await ctx.storage.store(blob);

const { fileId } = await supportAgent.addFile(ctx, {
  storageId,
  hash,
  filename: "receipt.pdf",
  mediaType: "application/pdf",
});

await supportAgent.messages.save(ctx, {
  threadId,
  userId,
  message: {
    message: {
      author: { type: "user", userId },
      content: [
        { type: "text", text: "Please read this receipt." },
        {
          type: "file",
          fileId,
          mediaType: "application/pdf",
          filename: "receipt.pdf",
        },
      ],
    },
    fileIds: [fileId],
  },
});

// Agent owns dedupe, refcounts, and cleanup.
await supportAgent.getFile(ctx, { fileId });
After: App-owned files
// App code: store bytes wherever this app stores files.
const blob = new Blob([bytes], { type: mediaType });
const storageId = await ctx.storage.store(blob);

const appFileId = await ctx.db.insert("files", {
  storageId,
  ownerId: userId,
  hash,
  mediaType: "application/pdf",
  filename: "receipt.pdf",
  createdAt: Date.now(),
});

await supportAgent.saveMessage(ctx, {
  threadId,
  userId,
  message: {
    author: { type: "user", userId },
    content: [
      { type: "text", text: "Please read this receipt." },
      {
        type: "file",
        fileId: appFileId,
        mediaType: "application/pdf",
        filename: "receipt.pdf",
      },
    ],
  },
});

// App owns authorization, dedupe, URLs, and cleanup.

Optional context composition

Resolve app file refs before model execution
const loadFileContext: AgentContextLoader = async (ctx, { recentMessages }) => {
  const fileIds = recentMessages.flatMap((message) =>
    message.message.content.flatMap((part) =>
      part.type === "file" && part.fileId ? [part.fileId] : [],
    ),
  );

  const files = await ctx.runQuery(internal.files.describeForAgent, {
    fileIds,
  });

  return files.map((file) => ({
    type: "text",
    name: "file",
    text: `${file.filename}: ${file.summary}`,
    metadata: { fileId: file._id, mediaType: file.mediaType },
  }));
};

await supportAgent.runs.execute(ctx, {
  runId,
  context: [loadFileContext],
});

Boundary decisions

Concern Owner after cleanup Reason
Blob storage App Apps choose Convex storage, external storage, generated files, or URLs.
Dedupe and hashes App Dedupe policy depends on tenant, auth, retention, and product semantics.
File authorization App Agent cannot know which users may read or delete app files.
Message file parts Agent Agent stores typed message content exactly as the app supplied it.
Cleanup App Retention and deletion need app policy, not Agent guesses.
Retrieval and memory App RAG, embeddings, and memory use app-owned components exposed through context loaders.
Rate limits and API keys App Quota, billing, tenant policy, and auth happen before Agent APIs run.
Run event storage Stream Stream owns ordered events, cursors, idempotent tail appends, and HTTP serving.
WebSocket delta machinery Removed Agent should not keep streamDeltas, streamingMessages, or custom SSE/WebSocket protocol code.
Framework protocols Adapters React, Vercel, TanStack, and AI SDK map to Agent runs without shaping core.

Implementation cleanup completed

  • Delete Agent's `files` table.
  • Delete `agent.addFile`, `agent.useExistingFile`, and `agent.getFile`.
  • Remove `messages.fileIds` and refcount updates from message mutations.
  • Keep file content parts as plain app-owned references.
  • Project tool-call state from run events into bounded internal rows.
  • Keep RAG, rate limits, API keys, workflows, and UI adapters outside Agent core.
  • Schema tests assert Agent no longer owns file storage.

What stays in Agent core

Agent still owns durable runs, thread messages, tool approval APIs, model-agnostic event payloads, and materialized Agent messages. Everything else composes through app functions or adapter packages.
